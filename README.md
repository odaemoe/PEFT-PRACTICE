# PEFT для анализа тональности — технологическая практика

Программная реализация и экспериментальное исследование параметро-эффективных
методов дообучения (PEFT) трансформерной модели **RoBERTa-base** для задачи
бинарного анализа тональности текста.

Проект сравнивает методы **Full Fine-Tuning, LoRA (r=4/8/16), IA3, Prompt Tuning**
на двух датасетах (**IMDb** и **SST-2**) по качеству и ресурсоёмкости.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `config.py` | все гиперпараметры: модель, датасеты, методы, LR |
| `data_utils.py` | загрузка, унификация и токенизация датасетов |
| `models.py` | инициализация модели и PEFT-конфигураций |
| `train_eval.py` | цикл обучения, оценка, замеры времени/VRAM |
| `metrics.py` | сводные таблицы и графики |
| `main.py` | точка входа для запуска экспериментов |
| `demo.ipynb` | демонстрационный ноутбук (Google Colab) |

## Запуск в Google Colab (рекомендуется)

1. Открыть `demo.ipynb` в Colab.
2. Среда выполнения → Сменить среду выполнения → **GPU (T4)**.
3. Выполнить ячейки по порядку (или «Выполнить всё»).

## Запуск локально (нужен GPU)

```bash
pip install -r requirements.txt

python main.py --quick          # быстрый smoke-test на малых данных
python main.py                  # полный прогон (все датасеты и методы)
python main.py --datasets SST-2 # только один датасет
python main.py --save-adapters  # дополнительно сохранить обученные адаптеры
```

## Результаты

После запуска в папке `results/` появляются:

- `summary_<датасет>.csv` / `.md` — сводная таблица метрик;
- `curves_<датасет>.png` — кривые обучения всех методов;
- `compare_<датасет>.png` — диаграммы сравнения качества и ресурсов.

Обученные адаптеры (при `--save-adapters`) сохраняются в папку `adapters/`.

## Как менять эксперимент

Всё настраивается в `config.py`:

- `TRAIN_SIZE` / `TEST_SIZE` — объём подвыборок;
- `METHODS` — набор методов и их learning rate;
- `DATASETS` — список датасетов.

> **Важно про learning rate.** У IA3 и Prompt Tuning очень мало обучаемых
> параметров, поэтому им нужен LR на 2 порядка больше, чем у Full FT/LoRA.
> В `config.py` для каждого метода задан свой LR — без этого IA3 и Prompt
> Tuning не обучаются (именно эта проблема возникала в исходной курсовой).
